# Project Map

Living index of what exists in this repo and what each file is for.
Update this alongside code changes — don't let it drift.

> **Stack note:** this repo runs **Next.js 16 (App Router) + Tailwind CSS v4
> + React 19**, not the Next 14 / Tailwind v3 stack BUILD_SPEC.md assumes.
> Tailwind v4 is CSS-first: design tokens live in `@theme` inside
> `globals.css`, not in `tailwind.config.ts`. See `AGENTS.md`.

## Config

| File | Purpose |
|---|---|
| `tailwind.config.ts` | Content-scanning globs only (`./src/**/*.{ts,tsx}`). No theme data — v4 keeps that in CSS. Wired into `globals.css` via `@config`. |
| `postcss.config.mjs` | Registers the `@tailwindcss/postcss` plugin. |
| `tsconfig.json` | `@/*` path alias → `./src/*`. |

## Design-system atoms — `src/components/ui/`

Generic, content-agnostic building blocks reused across every section.

| File | Purpose |
|---|---|
| `Section.tsx` | Page-section wrapper: 1200px max-width, 32px horizontal / ~90px vertical padding (64px on mobile). `bg` prop maps to §2 background tokens. `as="section" \| "footer"` picks the outer tag (Footer needs a real `<footer>`, not a `<section>`). |
| `SectionLabel.tsx` | Small uppercase violet label above section titles (0.07em tracking). |
| `Button.tsx` | `variant: "primary" \| "ghost" \| "dark"`. Renders `next/link` when `href` is passed, a real `<button>` otherwise (form submit). |
| `Chip.tsx` | 44px tinted icon chip for card grids (Services, Values). `tone` covers violet + the four supporting tints. |
| `TodoBadge.tsx` | Dashed-border red chip flagging an unconfirmed value (stat numbers, testimonial quotes, contact details) per §3 honesty guardrails — reused across Stats, Testimonials, Cases, TechStack, Contact rather than repeating the same markup each time. |
| `GhostWord.tsx` | Faint oversized background word (`aria-hidden`, `pointer-events-none`, `font-display font-extrabold`, responsive `text-[4rem]→lg:text-[7rem]`) sitting behind a section's eyebrow/heading — `tone="light"` (`text-ink/[0.04]`, default) for light-bg sections, `tone="dark"` (`text-white/[0.05]`) for dark ones. Requires its section to pass `className="relative overflow-hidden"` to `<Section>` so long phrases bleed off and clip at the section edge instead of causing horizontal overflow on narrow viewports. Used by every section that has a short eyebrow/name (Stats, Values, Cases, Industries, TechStack, FAQ, Contact, Testimonials, Discovery, Trusted, Pillars) — see each section's row below for its exact ghost text. **Not** used by `Hero.tsx` (its H1 is already the size a ghost word would be, so the effect would just clash) or `Services.tsx` (which has its own pre-existing, separately-pasted `.svc-ghost` CSS implementation of the same idea — see that row). |
| `index.ts` | Barrel export for the atoms above. |

| `Pillars.tsx` | S4 — Hero pillars, restyled as an inline dark band that flows directly out of the Hero: same `hero-gradient` bg token, no top radius, `rounded-b-[40px]` only on the bottom so the two sections read as one continuous dark block that curves into `Trusted` below. 3 equal columns (single column on mobile, `border-t`/`sm:border-l` divider at `border-white/[0.14]` between them) each holding one short white `font-display` headline (~19px), plus a `GhostWord tone="dark"` ("OUR PILLARS") behind them since this band has no eyebrow of its own. | `Values.tsx` | S7 — Values ("What we stand for", replaces the SoluLab awards carousel) on `white` with a `GhostWord` ("WHY TRESTO") behind the eyebrow: 5 equal, lightly-bordered (`border-[0.5px] border-border`) cards — Trust/Ownership/Quality/Transparency/Support — each a 46px `bg-lilac-card` icon chip + title + one-line description, `grid-cols-2 lg:grid-cols-5`. ## Shared utilities

| File | Purpose |
|---|---|
| `src/lib/cn.ts` | Tiny classname-join helper (filters falsy values, joins with a space). No `clsx`/`tailwind-merge` dependency added. |
| `src/lib/usePrefersReducedMotion.ts` | `(prefers-reduced-motion: reduce)` media-query hook, `useState(false)` initial + `useEffect` sync so SSR/hydration never mismatch. Shared by `Navbar.tsx`, `Hero.tsx`, `Pillars.tsx`, `Trusted.tsx`, and `Stats.tsx`. Not used by `Services.tsx` — its pasted-in version does its own inline `window.matchMedia("(prefers-reduced-motion: reduce)")` check per scroll frame instead. |
| `src/lib/useMediaQuery.ts` | Generic `(query: string)` media-query hook — same `useState(false)` + `useEffect` SSR-safe pattern as `usePrefersReducedMotion.ts`, just parameterized. Currently unused (originally added for an earlier `Services.tsx` version); left in place rather than deleted. |
| `src/types/index.ts` | Shared content shapes (`NavLink`, `Service`, `Stat`, `StatsSectionContent`, `Testimonial`, `FAQItem`, `HeroContent`, `SiteConfig`, etc.). `HeroContent` gained a `bullets: string[]` field (missing from the original shape) to cover S3's 3 value props. Now consumed by `Navbar.tsx` (`NavLink`), `Hero.tsx` (`HeroContent`), `Stats.tsx` (`Stat`/`StatsSectionContent`), `Testimonials.tsx` (`Testimonial`), `FAQ.tsx` (`FAQItem`). Sections with a shape that didn't fit an existing type (Pillars, Cases, Industries, Values, TechStack, and now Services) define a small local interface instead of stretching a shared one. `Service` is currently unused — `Services.tsx` defines its own local `ServiceCard` interface (icon as a `LucideIcon` component, not a string key) rather than importing from `@/types`; left in place rather than deleted. |
